# 논문 구조 이해하기

Abstract, Method, Result를 빠르게 파악하는 법

<!-- script: 이번 강의에서는, 학술 논문의 기본 구조를 이해하고, 각 섹션에서 어떤 정보를 찾아야 하는지 배워보겠습니다. -->

---

# 논문의 기본 구조

- Abstract: 연구 전체를 압축한 요약문
- Introduction: 연구 배경, 문제 제기, 목적
- Methods: 연구 설계, 데이터 수집, 분석 방법
- Results: 실험 결과와 데이터 분석 내용

<!-- script: 논문은 크게 네 부분으로 나뉩니다. Abstract은 전체 요약이고, Introduction은 왜 이 연구를 했는지, Methods는 어떻게 했는지, Results는 무엇을 발견했는지를 담고 있습니다. -->

---

# 빠르게 읽는 전략

- 1단계: Abstract로 전체 흐름 파악
- 2단계: Figures와 Tables 먼저 확인
- 3단계: Conclusion에서 핵심 기여 확인
- 4단계: 필요한 섹션만 깊이 읽기

<!-- script: 모든 논문을 처음부터 끝까지 읽을 필요는 없습니다. Abstract을 먼저 읽고, 그림과 표를 확인한 뒤, 결론에서 핵심 기여를 파악하세요. 그 다음에 필요한 부분만 깊이 읽으면 됩니다. -->

---

# 효과적인 요약 프롬프트

- 역할 지정: '논문 리뷰어로서 다음 논문을 분석해줘'
- 구조화 요청: '연구 목적, 방법, 결과, 한계를 정리해줘'
- 분량 지정: '300자 이내로 요약해줘'

<!-- script: AI에게 논문 요약을 맡길 때는, 역할을 지정하고, 출력 구조를 명확히 하며, 분량을 제한하는 것이 중요합니다. 이렇게 하면 일관된 품질의 요약을 얻을 수 있습니다. -->

---

# 요약 품질 높이기

- 원문 Abstract과 AI 요약 비교 검증
- 핵심 수치와 통계값이 정확한지 확인
- AI가 생성한 요약에 없는 원문 내용 보충
- 여러 번 질문하며 깊이 있는 이해 도달

<!-- script: AI 요약을 그대로 사용하면 안 됩니다. 반드시 원문과 비교하고, 핵심 수치가 맞는지 확인하세요. AI에게 추가 질문을 하면서, 논문에 대한 이해를 깊게 만들어 가는 것이 좋습니다. -->

# 논리적 글 구조 설계

서론-본론-결론의 탄탄한 뼈대 만들기

<!-- script: 이번 강의에서는, 서술형 글의 기본인 서론, 본론, 결론 구조를 설계하는 방법을 배워보겠습니다. -->

---

# 서론-본론-결론 프레임워크

- 서론: 주제 소개, 배경, 글의 목적 제시
- 본론: 근거와 사례를 논리적으로 전개
- 결론: 핵심 요약, 시사점, 향후 과제

<!-- script: 서술형 글은 서론에서 독자의 관심을 끌고, 본론에서 근거를 제시하며, 결론에서 핵심을 정리하는 구조를 따릅니다. 이 기본 구조를 먼저 잡아야, 읽기 쉬운 글이 됩니다. -->

---

# 아웃라인 작성법

- 핵심 주장(thesis statement)을 먼저 정의
- 뒷받침할 근거 3~5개를 나열
- 각 근거에 대한 사례나 데이터 배치
- 논리적 흐름을 점검하고 순서 조정

<!-- script: 글을 바로 쓰기보다, 아웃라인을 먼저 작성하세요. 핵심 주장을 정하고, 이를 뒷받침할 근거를 나열한 뒤, 사례와 데이터를 배치합니다. 이 과정을 거치면 글의 논리적 흐름이 훨씬 좋아집니다. -->

# AI 초안 생성과 사람의 편집

AI는 조수, 최종 판단은 사람이

<!-- script: 이번 강의에서는, AI로 초안을 빠르게 생성하고, 사람이 편집하여 완성도를 높이는 워크플로우를 배워보겠습니다. -->

---

# AI 초안 생성 전략

- 아웃라인을 AI에게 제공하고 섹션별 초안 요청
- 톤과 독자 수준을 프롬프트에 명시
- 한 번에 전체를 쓰지 말고 단락별로 생성
- 여러 버전을 생성하고 가장 좋은 부분 선택

<!-- script: AI에게 초안을 맡길 때는, 아웃라인을 먼저 제공하고, 섹션별로 나눠서 요청하세요. 톤과 독자 수준을 명시하면 더 적합한 초안을 얻을 수 있습니다. 한 번에 완벽한 글을 기대하지 말고, 여러 버전 중 좋은 부분을 조합하세요. -->

---

# 사람의 편집 체크리스트

- 사실 관계가 정확한지 원문과 대조
- 본인의 목소리와 관점이 반영되어 있는지 확인
- 불필요한 반복이나 모호한 표현 수정
- 인용과 출처가 올바르게 표기되었는지 검수

<!-- script: AI가 생성한 초안을 반드시 사람이 편집해야 합니다. 사실 관계를 확인하고, 본인의 관점을 반영하며, 모호한 표현을 다듬으세요. 특히 인용과 출처가 정확한지 꼼꼼히 확인하는 것이 중요합니다. -->

---

# AI 환각 검증하기

AI가 만들어낸 거짓 정보를 걸러내는 법

<!-- script: 이번 강의에서는, AI가 사실처럼 생성하지만 실제로는 틀린 정보, 즉 환각 현상을 식별하고 검증하는 방법을 배워보겠습니다. -->

---

# AI 환각이란?

- 존재하지 않는 논문이나 저자를 만들어내는 현상
- 실제와 다른 통계 수치를 자신있게 제시
- 그럴듯하지만 사실이 아닌 내용을 생성
- 출처를 요청하면 가짜 URL을 만들어내기도 함

<!-- script: AI 환각이란, AI가 사실이 아닌 정보를 마치 사실인 것처럼 생성하는 현상입니다. 존재하지 않는 논문을 인용하거나, 틀린 통계를 자신있게 말하기도 합니다. 이런 환각은 학술 글쓰기에서 매우 위험합니다. -->

---

# 환각 검증 방법

- AI가 언급한 논문 제목을 Google Scholar에서 직접 검색
- 통계 수치는 반드시 원문 데이터와 대조
- Perplexity로 출처가 있는 정보인지 교차 확인
- 핵심 주장에 대해 '출처를 알려줘'라고 재질문

<!-- script: 환각을 검증하는 가장 확실한 방법은, AI가 언급한 논문을 직접 검색해보는 것입니다. 통계 수치도 원문과 반드시 대조하세요. Perplexity 같은 검색 기반 AI를 활용하면, 출처가 있는 정보인지 빠르게 확인할 수 있습니다. -->

---

<!-- +qa -->
- q: "AI가 알려준 논문이 검색해도 안 나와요"
  a: "높은 확률로 AI가 만들어낸 가짜 논문입니다. AI는 실제로 존재하지 않는 논문 제목과 저자를 조합해서 생성하는 경우가 있습니다. 반드시 Google Scholar나 DOI 검색으로 실존 여부를 확인하세요."
- q: "팩트체크에 시간이 너무 오래 걸려요"
  a: "핵심 주장과 통계 수치 위주로 우선순위를 정해서 검증하세요. 모든 문장을 다 확인할 필요는 없습니다. Perplexity를 활용하면 출처 확인 시간을 크게 줄일 수 있습니다."

# 출처 추적 워크플로우

# 파이프라인 설계

- 1단계: Google Sheets에 논문 목록과 PDF 링크 정리

# API 활용 팁

- 토큰 제한을 고려해 논문을 섹션별로 분할 입력
- 시스템 프롬프트로 출력 형식을 고정
- 비용 관리를 위해 GPT-4o-mini 등 경량 모델 활용
- 에러 처리와 재시도 로직 구현

<!-- script: API를 사용할 때는, 토큰 제한을 고려해 논문을 섹션별로 나눠서 입력하세요. 시스템 프롬프트로 출력 형식을 고정하면 일관된 결과를 얻을 수 있습니다. 비용 절감을 위해 경량 모델을 활용하는 것도 좋은 전략입니다. -->
