# 왜 통화 녹음이 필요한가?

- 고객 요구사항을 빠짐없이 기록 → 견적 누락 방지
- 통화 후 기억에 의존하면 항목·수량 오류 발생
- 녹음 데이터가 있어야 AI 분석이 가능
- 분쟁 발생 시 근거 자료로 활용 가능

<!-- script: 먼저, 왜 통화를 녹음해야 하는지 짚어볼게요. 고객과 통화하면서 견적에 필요한 항목을 메모하다 보면, 빠뜨리는 경우가 많습니다. 녹음 데이터가 있으면, AI가 자동으로 분석해서 항목을 추출할 수 있고요, 나중에 분쟁이 생겼을 때도 근거 자료가 됩니다. -->

---

# 녹음 도구 비교와 설정

- Otter.ai: 영어 통화에 강점, 실시간 트랜스크립션 지원
- Clova Note: 한국어 인식률 우수, 네이버 계정으로 바로 사용
- Zoom: 화상 미팅 겸용, 클라우드 녹음 자동 저장
- 스마트폰 기본 녹음 앱도 활용 가능 (수동 변환 필요)

<!-- script: 녹음 도구를 비교해볼게요. Otter.ai는 영어 통화에 특히 강하고요, 실시간으로 텍스트 변환이 됩니다. Clova Note는 한국어 인식률이 뛰어나서, 국내 통화에 적합합니다. Zoom은 화상 미팅과 함께 사용할 때 편리하고요, 클라우드에 자동 저장됩니다. -->

---

# 녹음 시 주의사항

- 통화 녹음 전 상대방 동의 필수 (개인정보보호법)
- 녹음 시작 시 '이 통화는 녹음됩니다' 안내 권장
- 조용한 환경에서 통화 → STT 정확도 향상
- 녹음 파일은 WAV 또는 MP3 형식으로 저장

<!-- script: 녹음할 때 주의할 점도 있는데요. 법적으로, 상대방에게 녹음 사실을 알려야 합니다. 그리고 조용한 환경에서 통화하면, 음성 인식 정확도가 훨씬 올라갑니다. 파일 형식은 WAV나 MP3로 저장하시면 됩니다. -->

# STT(음성→텍스트) 변환

녹음 파일을 분석 가능한 텍스트로 바꾸기

<!-- script: 이번에는, 녹음된 통화 파일을, 텍스트로 변환하는 방법을 알아보겠습니다. STT란 Speech to Text의 약자로, 음성을 글자로 바꿔주는 기술입니다. -->

---

# STT 변환 프로세스

- 1단계: 녹음 파일 업로드 (Otter.ai 또는 Clova Note)
- 2단계: AI가 자동으로 음성 인식 및 텍스트 변환
- 3단계: 화자 분리 (고객 vs 나) 자동 구분
- 4단계: 변환된 텍스트 검토 및 오류 수정

<!-- script: STT 변환 과정은 간단합니다. 먼저 녹음 파일을 도구에 업로드하면, AI가 자동으로 음성을 인식해서 텍스트로 바꿔줍니다. 화자 분리 기능이 있어서, 고객의 말과 나의 말을 구분해주고요. 마지막으로 변환된 텍스트를 검토해서, 잘못 인식된 부분을 수정하면 됩니다. -->

---

# 변환 품질 높이는 팁

- 녹음 품질이 좋을수록 인식률 향상 (잡음 제거 중요)
- 전문 용어가 많으면 사전에 용어 사전 등록
- 변환 후 핵심 키워드 위주로 빠르게 검토
- 화자별 발화를 구분해두면 이후 분석이 쉬워짐

<!-- script: 변환 품질을 높이려면, 녹음할 때 잡음을 최소화하는 게 가장 중요합니다. 그리고 업종별 전문 용어가 있다면, 미리 용어 사전에 등록해두면 인식률이 올라갑니다. 변환 후에는 전체를 다 읽기보다, 핵심 키워드 위주로 빠르게 검토하세요. -->

---

# 텍스트 정리와 저장

- 변환된 텍스트를 Google Docs 또는 Notion에 저장
- 통화 일시, 고객명, 주요 키워드 태그 추가
- 견적 관련 발화 부분 하이라이트 표시
- 정리된 스크립트를 다음 단계(AI 분석)에 전달

<!-- script: 변환이 끝나면, 텍스트를 깔끔하게 정리해서 저장합니다. Google Docs나 Notion에 저장하면서, 통화 일시와 고객명을 기록해두세요. 견적과 관련된 부분은 하이라이트 해두면, 다음 단계에서 AI가 분석할 때 훨씬 효율적입니다. -->

# 통화 내용에서 핵심 요구사항 추출

# AI 분석이 추출하는 정보

- 고객이 요청한 제품/서비스 항목
- 수량, 규격, 사양 등 상세 스펙
- 납기일, 배송 조건 등 부대 조건
- 특별 요청사항 및 할인 관련 언급

# ChatGPT 프롬프트 설계

- 역할 지정: '당신은 견적서 작성 전문가입니다'
- 출력 형식 지정: JSON 또는 표 형태로 구조화 요청
- 추출 항목 명시: 품목명, 수량, 단위, 예상 단가, 비고
- 맥락 제공: 업종, 주요 취급 품목 정보 포함

<!-- script: ChatGPT를 잘 활용하려면, 프롬프트 설계가 중요합니다. 먼저 역할을 지정해서, 견적서 작성 전문가로 설정하고요. 출력 형식을 JSON이나 표 형태로 지정하면, 나중에 자동 처리가 쉬워집니다. 그리고 추출할 항목을 명확히 나열하고, 업종 정보도 함께 제공하면 정확도가 올라갑니다. -->

---

# 추출 결과 검증 체크리스트

- 통화에서 언급된 모든 항목이 추출되었는지 확인
- 수량과 단위가 정확한지 대조
- 고객의 특별 요청이 반영되었는지 점검
- 누락 항목은 수동으로 추가 보완

<!-- script: AI가 추출한 결과를, 반드시 검증해야 합니다. 통화에서 언급된 모든 항목이 빠짐없이 추출되었는지, 수량과 단위가 맞는지 확인하세요. 고객의 특별 요청도 빠지지 않았는지 점검하고요, 누락된 게 있으면 수동으로 추가하면 됩니다. -->

# 항목·수량·단가 자동 파싱

추출된 데이터를 견적서 형식으로 구조화하기

<!-- script: 이번에는, AI가 추출한 정보를, 견적서에 바로 넣을 수 있는 형태로 구조화하는 방법을 알아보겠습니다. 파싱이란, 데이터를 원하는 형식으로 분해하고 정리하는 과정입니다. -->

---

# 구조화 데이터 형식

- 품목명 | 규격/사양 | 수량 | 단위 | 단가 | 공급가 | 비고
- JSON 형식으로 출력하면 자동화 연동이 쉬움
- 단가 정보가 없으면 기존 가격표에서 자동 매칭
- 부가세 별도/포함 여부도 함께 기록

<!-- script: 구조화된 데이터는, 품목명부터 비고까지, 견적서의 각 열에 대응하는 형태로 정리합니다. JSON 형식으로 출력하면, 나중에 자동화 도구와 연동하기 편하고요. 통화에서 단가가 안 나왔다면, 기존 가격표에서 자동으로 매칭할 수 있습니다. -->

---

# Google Sheets로 데이터 정리

- ChatGPT 출력을 Google Sheets에 붙여넣기
- 수식으로 공급가(수량 × 단가) 자동 계산
- 부가세(10%) 자동 산출 및 합계 계산
- 단가 데이터베이스 시트와 VLOOKUP 연동

<!-- script: ChatGPT에서 추출한 데이터를, Google Sheets에 정리합니다. 수량 곱하기 단가로 공급가를 자동 계산하고요, 부가세 10%도 수식으로 자동 산출합니다. 기존 단가 데이터베이스가 있다면, VLOOKUP으로 연동하면 단가 입력도 자동화할 수 있습니다. -->

---

# 파싱 자동화 실습

- ChatGPT API를 활용한 자동 파싱 스크립트 작성
- 통화 텍스트 입력 → 구조화 JSON 출력 파이프라인
- 오파싱(잘못된 파싱) 감지 및 자동 재시도 로직
- 파싱 결과를 바로 스프레드시트에 자동 입력

<!-- script: 한 단계 더 나아가서, 파싱을 완전 자동화할 수도 있습니다. ChatGPT API를 활용하면, 통화 텍스트를 넣으면 구조화된 JSON이 바로 나오는 파이프라인을 만들 수 있고요. 잘못 파싱된 경우를 감지해서 자동으로 재시도하는 로직도 추가할 수 있습니다. -->

# 견적서 필수 구성요소

- 헤더: 회사명, 로고, 사업자번호, 연락처
- 수신 정보: 고객사명, 담당자명, 연락처
- 견적 항목: 품목, 규격, 수량, 단가, 공급가, 세액
- 하단: 합계금액, 납품조건, 유효기간, 비고

<!-- script: 견적서에 반드시 들어가야 할 요소를 정리해볼게요. 헤더에는 우리 회사 정보가, 그 아래는 고객사 정보가 들어갑니다. 중간에 견적 항목 테이블이 오고요, 하단에는 합계 금액과 각종 조건을 기재합니다. -->

# Google Sheets 템플릿 만들기

- 셀 병합과 서식으로 전문적인 레이아웃 구성
- 수량 × 단가 = 공급가 자동 계산 수식 설정
- 부가세(10%) 및 총 합계 자동 산출
- 조건부 서식으로 금액이 큰 항목 강조 표시

<!-- script: Google Sheets에서 템플릿을 만들어볼게요. 셀 병합과 서식을 활용해서, 전문적인 레이아웃을 구성합니다. 수량 곱하기 단가 수식을 넣어두면, 데이터만 입력하면 금액이 자동 계산되고요. 부가세와 합계도 자동으로 나옵니다. -->

---

# 템플릿 활용 팁

- 업종별 견적서 템플릿을 여러 개 준비
- 자주 쓰는 품목은 드롭다운 목록으로 선택
- 견적 번호 자동 채번 (날짜 + 일련번호)
- 회사 직인 이미지를 미리 삽입해두기

<!-- script: 템플릿을 더 효과적으로 활용하는 팁을 알려드릴게요. 업종별로 여러 템플릿을 만들어두면 편하고요, 자주 쓰는 품목은 드롭다운으로 선택할 수 있게 설정하세요. 견적 번호는 날짜와 일련번호를 조합해서 자동으로 만들면, 관리가 훨씬 쉬워집니다. -->

# AI로 견적서 초안 자동 완성

# 자동 완성 프로세스

- 1단계: 파싱된 JSON 데이터를 Google Sheets에 자동 입력
- 2단계: 수식으로 금액 자동 계산 (공급가, 세액, 합계)
- 3단계: ChatGPT로 비고란, 납품조건 등 문구 자동 생성
- 4단계: 최종 검토 후 확정

<!-- script: 자동 완성은 네 단계로 진행됩니다. 먼저, 파싱된 데이터가 Google Sheets 템플릿에 자동으로 입력되고요, 수식이 금액을 계산합니다. 그 다음 ChatGPT가 비고란이나 납품조건 같은 문구를 생성하고요, 마지막으로 사람이 검토해서 확정합니다. -->

---

# ChatGPT 문구 자동 생성

- 납품 조건: 통화 내용 기반 맞춤 조건 문구 작성
- 유효기간: 업종 관행에 맞는 기간 자동 설정
- 특이사항: 고객 특별 요청을 비고란에 반영
- 인사말: 고객사에 맞는 견적서 커버 레터 생성

<!-- script: ChatGPT가 자동으로 생성하는 문구를 살펴볼게요. 통화 내용을 바탕으로 납품 조건을 맞춤 작성하고요, 유효기간도 업종 관행에 맞게 설정합니다. 고객의 특별 요청은 비고란에 반영하고, 견적서와 함께 보낼 커버 레터도 만들 수 있습니다. -->

---

# 견적서 검토 포인트

- 항목 누락 여부: 통화 스크립트와 대조 확인
- 금액 정확성: 단가 × 수량 계산 오류 점검
- 고객 정보: 회사명, 담당자명 오타 확인
- 조건 적절성: 납기일, 결제조건이 합의 내용과 일치하는지

<!-- script: 견적서를 보내기 전에, 반드시 검토해야 할 포인트가 있습니다. 항목이 빠지지 않았는지, 금액 계산에 오류가 없는지, 고객 정보에 오타가 없는지, 그리고 납기일과 결제조건이 통화 내용과 일치하는지 꼼꼼히 확인하세요. -->

# 전체 자동화 흐름

- 통화 종료 → Otter.ai/Clova Note에서 자동 텍스트 변환
- 변환된 스크립트 → ChatGPT API로 자동 파싱
- 파싱 결과 → Google Sheets 견적서 템플릿에 자동 입력
- 견적서 완성 → PDF 변환 → 이메일 자동 발송

<!-- script: 전체 자동화 흐름을 정리해볼게요. 통화가 끝나면 STT 도구가 자동으로 텍스트를 만들고, ChatGPT API가 핵심 정보를 파싱합니다. 그 결과가 Google Sheets 템플릿에 자동으로 들어가서 견적서가 완성되고요, PDF로 변환해서 이메일까지 자동 발송됩니다. -->

# Zapier로 워크플로우 구축

- 트리거: 새 녹음 파일 생성 시 자동 시작

- 액션 2: ChatGPT API 호출 → 항목 파싱

<!-- script: 자동화를 안정적으로 운영하려면, 오류 처리가 중요합니다. 각 단계에서 실패하면 Slack이나 이메일로 알림이 오게 설정하고요. 파싱 결과의 신뢰도가 낮으면, 자동으로 수동 검토를 요청하게 만들 수 있습니다. Zapier 대시보드에서 전체 실행 이력도 확인할 수 있습니다. -->
