# 자동화의 기본 구조

![자동화의 기본 구조](https://www.vibecamp.us/course-images/level2-automation-flow.svg)

*입력(Trigger) → 처리(AI+자동화) → 출력(Action)*

---
# 자동화란 무엇인가

사람이 반복하는 일을 기계가 대신하는 원리를 이해합니다.

---

# 자동화의 정의

- 같은 절차를 매번 동일하게 실행하는 것
- 사람의 판단이 필요 없는 단계를 기계에 위임
- 목표: 시간 절약 + 실수 감소

---

# 일상 속 자동화 예시

- 이메일 자동 필터링 (Gmail 규칙)
- 스프레드시트 함수로 합계 자동 계산
- 매일 아침 날씨 알림 봇

---

# AI 자동화가 다른 점

- 기존 자동화: 정해진 규칙만 실행
- AI 자동화: 텍스트를 이해하고 판단까지 포함
- 예) 이메일 내용을 읽고 '긴급'인지 분류

---

# Level 2에서 다루는 범위

- 챗봇 밖으로 AI를 꺼내기
- 트리거 → AI 처리 → 결과 전달 흐름
- 복잡한 에이전트는 Level 3 이후에

---

# 핵심 정리

- 자동화 = 반복 절차의 기계 위임
- AI를 붙이면 '판단'까지 자동화 가능
- 이번 과정에서 직접 만들어 봅니다

---

<!-- +qa -->
- q: "프로그래밍을 못해도 자동화를 할 수 있나요?"
  a: "네. n8n 같은 노코드 도구를 쓰면 드래그 앤 드롭으로 워크플로우를 만들 수 있습니다. 코드 한 줄 없이 시작할 수 있어요."
- q: "자동화와 AI 에이전트의 차이는 뭔가요?"
  a: "자동화는 미리 정해 놓은 흐름을 따릅니다. 에이전트는 스스로 판단해서 도구를 선택합니다. Level 2에서는 자동화에 집중하고, 에이전트는 Level 3 이후에 다룹니다."

# 반복 작업 찾아내기

내 하루를 관찰해서 자동화할 수 있는 업무를 골라 봅니다.

---

# 반복 작업의 특징

- 매일 또는 매주 같은 절차를 따른다
- 판단보다는 복사·붙여넣기가 많다
- 실수하면 되돌리기 어렵지 않다

---

# 업무 관찰 방법

- 하루 동안 한 일을 30분 단위로 적어 보기
- 같은 앱을 오가며 복사하는 작업에 형광펜 표시
- 주 3회 이상 반복되면 자동화 후보

---

# 자동화 후보 체크리스트

- 입력이 명확한가?
- 처리 방식이 매번 같은가?
- 결과를 보낼 곳이 정해져 있는가?
- 예외가 적은가?

---

# 예시: 자동화 가능한 업무들

- 매일 아침 뉴스 요약 → 슬랙 전달
- 고객 문의 접수 → 카테고리 분류 → 시트 기록
- 회의록 받기 → 핵심 요약 → 팀 채널 공유

---

# 실습 과제

- 내 업무에서 반복 작업 3개 적어 보기
- 각각에 체크리스트 적용해서 점수 매기기
- 가장 점수 높은 1개를 이 과정의 프로젝트로 선정

---

  a: "Level 2에서는 예외가 적은 것부터 시작하는 게 좋습니다. 예외 처리는 Level 3에서 AI 에이전트를 쓰면 더 잘 다룰 수 있습니다."

# 입력-처리-출력 사고법

모든 자동화를 세 단계로 분해하는 사고 프레임을 배웁니다.

---

# 세 단계 프레임

- 입력(Input): 데이터가 들어오는 곳 — 폼, 이메일, 웹훅 등
- 처리(Process): 데이터를 변환하는 곳 — 요약, 분류, 포맷 변경
- 출력(Output): 결과를 보내는 곳 — 슬랙, 시트, 이메일

---

# 왜 이렇게 나누는가

- 복잡해 보이는 작업도 세 칸으로 정리하면 단순해진다
- 각 단계를 독립적으로 테스트할 수 있다
- n8n 노드 배치가 이 구조와 1:1로 대응된다

---

# 실전 예시 1: 뉴스 요약 봇

- 입력: RSS 피드에서 새 기사 감지
- 처리: AI로 3줄 요약 생성
- 출력: 슬랙 채널에 메시지 전송

---

# 실전 예시 2: 고객 문의 분류

- 입력: 구글 폼 제출
- 처리: AI가 문의 유형(환불/배송/기타) 분류
- 출력: 구글 시트에 행 추가 + 담당자에게 알림

---

# 직접 그려 보기

- 종이에 가로로 세 칸을 그린다
- 선정한 반복 작업을 각 칸에 채운다
- 빈 칸이 있으면 아직 정의가 부족한 것

---

# 핵심 정리

- 자동화 = 입력 → 처리 → 출력
- 이 프레임이 n8n 워크플로우의 뼈대가 된다
- 다음 스테이지에서 실제로 만들어 봅니다

---

<!-- +qa -->
- q: "처리 단계가 여러 개면 어떻게 하나요?"
  a: "처리를 세부 단계로 나눠도 됩니다. 예를 들어 '요약 → 번역 → 포맷 변경'처럼 체인으로 연결하면 됩니다. n8n에서는 노드를 순서대로 이으면 됩니다."
- q: "입력이 여러 곳에서 들어오면요?"
  a: "n8n에서는 트리거 노드를 여러 개 둘 수 있습니다. 또는 하나의 웹훅으로 통합해서 받는 방법도 있습니다."
- q: "출력을 여러 곳에 동시에 보낼 수 있나요?"
  a: "네. n8n에서 처리 노드 뒤에 분기를 만들면 슬랙과 이메일에 동시에 보낼 수 있습니다."

# n8n 환경 설정하기

# n8n이란

- 오픈소스 워크플로우 자동화 도구
- 노드를 연결해서 자동화를 만든다

---

# 설치 방법 3가지

- n8n Cloud: 가입만 하면 바로 사용 (추천)
- Docker: docker run -it --rm -p 5678:5678 n8nio/n8n
- npx: npx n8n (Node.js 필요)

---

# 첫 화면 둘러보기

- 왼쪽: 워크플로우 목록
- 가운데: 캔버스 (노드 배치 영역)
- 오른쪽: 노드 설정 패널
- 상단: 실행 버튼과 저장

---

# 핵심 개념: 노드

- 트리거 노드: 워크플로우를 시작하는 이벤트
- 액션 노드: 데이터를 처리하거나 보내는 동작
- 노드끼리 선으로 연결하면 데이터가 흐른다

---

# 실습: Hello World 워크플로우

- Manual Trigger 노드 추가
- Set 노드로 {"message": "안녕하세요"} 설정
- 실행 버튼 클릭 후 결과 확인

# 첫 자동화 만들기

구글 폼 응답을 자동으로 구글 시트에 정리합니다.

---

# 만들 워크플로우

- 입력: 웹훅으로 폼 데이터 수신
- 처리: 필요한 필드만 추출
- 출력: 구글 시트에 행 추가

---

# Step 1: 웹훅 트리거 설정

- Webhook 노드 추가
- Method: POST 선택
- 테스트 URL 복사해 두기

---

# Step 2: 데이터 가공

- Set 노드로 필요한 필드만 매핑
- 이름, 이메일, 메시지 세 필드 설정
- 표현식(Expression)으로 입력 데이터 참조

---

# Step 3: 구글 시트 연결

- Google Sheets 노드 추가
- Credential(인증) 연결
- Append Row 동작 선택 후 컬럼 매핑

---

# 테스트와 확인

- 웹훅 URL에 테스트 데이터 전송 (Postman 또는 curl)
- 구글 시트에 행이 추가되었는지 확인
- 오류 나면 실행 로그에서 어느 노드가 실패했는지 확인

---

# 핵심 정리

- 웹훅 → Set → Google Sheets, 세 노드로 끝
- 입력-처리-출력 프레임이 그대로 적용된다
- 다음 레슨에서 AI를 처리 단계에 넣어 봅니다

---

<!-- +qa -->
- q: "구글 시트 말고 노션에도 보낼 수 있나요?"
  a: "네. n8n에 Notion 노드가 있어서 같은 방식으로 연결하면 됩니다. 출력 노드만 바꾸면 됩니다."
- q: "웹훅 테스트가 어려우면 어떻게 하나요?"
  a: "n8n에 내장된 테스트 기능을 쓰거나, 브라우저에서 직접 URL을 호출해도 됩니다. curl이 편하면 curl -X POST -d '{"name":"test"}' URL 형태로 보내세요."

# AI 노드 연결하기

워크플로우 중간에 AI를 넣어 텍스트를 처리합니다.

---

# AI 노드란

- n8n의 OpenAI / HTTP Request 노드로 AI API 호출
- 입력 텍스트를 보내고 응답을 받는 구조
- 요약, 분류, 번역, 감정 분석 등에 활용

---

# OpenAI 노드 설정

- OpenAI 노드 추가 → Credential에 API Key 입력
- Model: gpt-4o-mini (비용 효율적)
- Prompt에 이전 노드 데이터를 표현식으로 삽입

---

# 프롬프트 설계 팁

- 역할을 명확히 지정: '당신은 이메일 분류 전문가입니다'
- 출력 형식을 지정: 'JSON으로 {category, summary} 형태로 답하세요'
- 예시 하나를 넣으면 정확도가 크게 올라간다

---

# 실습: 폼 → AI 요약 → 시트 기록

- 이전 레슨의 워크플로우에 AI 노드 삽입
- Set 노드 대신 OpenAI 노드로 메시지 요약
- 요약 결과를 구글 시트에 기록

---

# 결과 확인과 디버깅

- AI 응답이 예상과 다르면 프롬프트 수정
- JSON 파싱 오류 시 출력 형식 지시를 더 구체적으로
- 실행 로그에서 각 노드의 입출력 데이터 확인

---

# 핵심 정리

- AI를 워크플로우 '처리' 단계에 넣으면 판단이 가능해진다
- 프롬프트 품질이 자동화 품질을 결정한다
- 이제 실전 프로젝트에 적용해 봅니다

---

# 프로젝트 1: 이메일 자동 분류

수신 이메일을 AI로 분류하고 라벨을 붙입니다.

---

# 완성 워크플로우 개요

- 입력: Gmail 트리거 (새 이메일 수신)
- 처리: AI가 카테고리 분류 (문의/제안/스팸/기타)
- 출력: 구글 시트에 기록 + 긴급 건은 슬랙 알림

---

# Step 1: Gmail 트리거 설정

- Gmail Trigger 노드 추가
- Credential 연결 후 폴링 간격 설정
- 테스트용 이메일을 보내서 트리거 확인

---

# Step 2: AI 분류 노드

- OpenAI 노드에 분류 프롬프트 작성
- 프롬프트: '다음 이메일을 [문의, 제안, 스팸, 기타] 중 하나로 분류하고 한 줄 요약하세요'
- 출력 형식: JSON {category, summary, is_urgent}

---

# Step 3: 조건 분기와 출력

- IF 노드로 is_urgent 여부 분기
- 모든 건: 구글 시트에 기록
- 긴급 건만: 슬랙 채널에 추가 알림

---

# 테스트 시나리오

- 일반 문의 이메일 → 시트에만 기록 확인
- 긴급 이메일 → 시트 기록 + 슬랙 알림 확인
- 스팸 이메일 → 올바르게 분류되는지 확인

---

# 핵심 정리

- 트리거 → AI 분류 → 조건 분기 → 복수 출력
- IF 노드로 분기하면 한 워크플로우에서 여러 경로 처리 가능
- 이런 패턴을 다른 업무에도 그대로 복사할 수 있다

# 프로젝트 2: 폼 → AI → 슬랙

외부 폼 입력을 AI가 처리해서 팀 채널에 전달합니다.

---

# 완성 워크플로우 개요

- 입력: 웹훅으로 폼 데이터 수신 (고객 피드백)
- 처리: AI가 감정 분석 + 핵심 요약 + 답변 초안 생성
- 출력: 슬랙 채널에 포맷된 메시지 전송

---

# Step 1: 웹훅 + 데이터 검증

- Webhook 노드로 POST 요청 수신
- IF 노드로 필수 필드(이름, 내용) 존재 여부 검증
- 빈 데이터는 에러 응답 반환

---

# Step 2: AI 처리 (세 가지 작업)

- 감정 분석: positive / neutral / negative
- 핵심 요약: 1~2문장으로 압축
- 답변 초안: 고객에게 보낼 응답 생성

---

# Step 3: 슬랙 메시지 포맷

- Slack 노드로 메시지 전송
- Block Kit 형식으로 보기 좋게 구성
- 감정이 negative면 빨간 이모지 표시

---

# 확장 아이디어

- 구글 시트에도 동시에 기록
- negative 피드백은 별도 채널로 라우팅
- 주간 요약 보고서 자동 생성 (Cron 트리거)

---

# 핵심 정리

- 하나의 AI 호출로 여러 작업(분석+요약+생성)을 한번에 처리
- 슬랙 Block Kit으로 팀이 바로 읽을 수 있는 형태 전달
- 이 패턴은 고객 문의, 설문, 리뷰 분석 등에 재활용 가능

---

<!-- +qa -->
- q: "슬랙 대신 디스코드나 팀즈에도 보낼 수 있나요?"
  a: "네. n8n에 Discord, Microsoft Teams 노드가 있습니다. 슬랙 노드를 해당 노드로 교체하면 됩니다."
- q: "AI 호출 비용이 걱정됩니다."
  a: "gpt-4o-mini 기준으로 피드백 1건 처리 비용은 약 0.001달러 수준입니다. 하루 100건 처리해도 월 3달러 미만이라 부담이 적습니다."
- q: "Block Kit 작성이 어려워요."
  a: "Slack Block Kit Builder(app.slack.com/block-kit-builder)에서 시각적으로 만들 수 있습니다. 만든 JSON을 n8n 노드에 붙여넣으면 됩니다."

# Level 2의 한계와 다음 단계

자동화만으로 안 되는 것을 알고, Level 3을 준비합니다.

---

# Level 2에서 할 수 있게 된 것

- 반복 작업을 찾아내고 자동화 설계
- n8n으로 트리거 → AI 처리 → 출력 워크플로우 구축
- 이메일 분류, 폼 처리, 슬랙 알림 등 실전 적용

---

# 한계 1: 예외 처리의 어려움

- 미리 정의하지 않은 상황에 대응 불가
- IF 분기가 늘어나면 워크플로우가 복잡해진다
- 사람이 직접 예외 케이스를 계속 추가해야 한다

---

# 한계 2: AI가 도구를 직접 못 쓴다

- AI는 텍스트만 처리하고, 행동은 사람이 연결해야 한다
- 파일 다운로드, 웹 검색, API 호출을 AI가 스스로 못 한다
- 매번 새로운 노드를 사람이 추가해야 한다

---

# 한계 3: 단방향 흐름

- 데이터가 한 방향으로만 흐른다
- AI가 결과를 보고 다시 판단하는 루프가 없다
- 복잡한 작업에는 사람의 중간 개입이 필요하다

---

# Level 3에서 달라지는 것

- AI가 외부 도구(파일, 웹, API)를 직접 사용
- Function Calling으로 AI가 스스로 다음 행동을 결정
- 사람 개입 없이 복잡한 작업을 자율적으로 수행

---

# 다음 단계 준비

- Level 2 프로젝트를 하나 이상 완성하고 실제로 사용해 보세요
- 자동화 한계를 직접 느끼면 Level 3의 필요성이 체감됩니다
- Level 3 — 도구 사용형 AI에서 만나겠습니다!
